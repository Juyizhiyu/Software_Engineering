# 产品需求文档 (PRD) - AI 赋能供应链可视化分析系统

## 1. 产品概述

AI 赋能企业供应链可视化分析系统 Demo，面向供应链管理人员，提供全局经营总览、数据录入管理、业务分析、风险监控和 AI 智能分析能力，帮助决策者快速掌握供应链运行状态并做出数据驱动的决策。

* 目标用户：供应链总监、运营经理、数据分析师

* 核心价值：将分散的供应链数据（订单/库存/供应商/物流/成本/风险）统一可视化，结合 AI 分析能力提供智能洞察

## 2. 核心功能

### 2.1 用户角色

| 角色  | 登录方式                 | 核心权限     |
| --- | -------------------- | -------- |
| 管理员 | 账号密码登录（admin/123456） | 全部功能访问权限 |

### 2.2 功能模块

1. **登录页**：用户身份验证，进入系统
2. **全局总览**：核心经营指标一览、销售趋势、风险分布、库存预警、优选供应商、最近订单
3. **数据中心**：六类实体数据（订单/库存/供应商/物流/成本/风险）的录入与查看
4. **业务分析**：库存优先处理、供应商履约排名、物流异常、高成本记录四大维度
5. **风险中心**：所有待处理风险的集中展示与管理
6. **AI 工作台**：智能问答、需求预测、异常检测、供应商风险评分四大 AI 功能

## 3. 核心流程

### 3.1 用户登录流程

用户打开系统 → 输入账号密码 → 调用登录接口 → 成功后存储 Token → 跳转至全局总览页

### 3.2 数据录入流程

进入数据中心 → 选择实体类型 Tab → 填写表单 → 提交数据 → 刷新列表

### 3.3 AI 分析流程

进入 AI 工作台 → 选择功能 Tab → 输入参数 → 调用 AI 接口 → 展示分析结果

## 4. 用户界面设计

### 4.1 设计风格

* **主色调**：深蓝科技风（#409EFF 主色 + #1D1E2C 深色背景），搭配青绿色（#67C23A）作为成功/正向指标色

* **按钮风格**：Element Plus 默认圆角按钮，主操作使用实心填充，次要操作使用描边

* **字体**：系统默认字体栈（-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto 等），数据展示使用等宽字体

* **布局风格**：左侧固定侧边栏导航 + 右侧主内容区，卡片式内容组织

* **图标风格**：Element Plus Icons（线性图标），简洁统一

* **深色模式**：支持亮色/深色模式切换，使用 useDark() 管理，所有组件和图表均适配双主题

### 4.2 页面设计概览

| 页面名称   | 模块名称     | UI 元素                                |
| ------ | -------- | ------------------------------------ |
| 登录页    | 登录表单     | 居中卡片布局，品牌 Logo，用户名/密码输入框，登录按钮，深色渐变背景 |
| 全局总览   | 指标卡片     | 4 列统计卡片（带图标和数值），ECharts 图表区域，3 列信息列表 |
| 数据中心   | Tab + 表单 | 顶部 Tab 切换，左侧表单区域，右侧数据列表              |
| 业务分析   | 四象限表格    | 2x2 网格布局，每个象限一个数据表格                  |
| 风险中心   | 风险卡片     | 瀑布流卡片布局，风险等级色彩标识                     |
| AI 工作台 | Tab + 结果 | 顶部 4 个功能 Tab，输入区域 + 结果展示区域           |

### 4.3 响应式设计

* 桌面优先设计，最小支持 1280px 宽度

* 侧边栏可折叠，内容区自适应宽度

* 表格在窄屏下支持横向滚动

### 4.4 可维护性设计

* **样式模块化**：SCSS 变量集中管理颜色/间距/字体，混合宏封装常用样式模式

* **组件化**：每个 UI 模块独立为组件，通过 Props/Emits 通信

* **业务逻辑分离**：数据处理和 API 调用提取至 composable 和 service 层

* **主题扩展点**：CSS 变量覆盖机制，便于后续主题定制

* **布局灵活性**：使用 CSS Grid/Flexbox，便于调整页面元素排列
