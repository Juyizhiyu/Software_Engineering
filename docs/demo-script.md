# 演示脚本

## 1. 总览页

展示订单总量、销售额、库存总量、供应链风险、销售趋势、风险分布、库存预警、供应商排行和最近订单。说明筛选条件支持区域、日期和品类，页面会显示当前数据源和数据质量。

## 2. 数据中心

切换六类实体：订单、库存、供应商、物流、成本、风险。演示新增一条记录后列表刷新，说明后端 schema 与前端 fallback schema 保持一致。

## 3. 业务分析

展示库存缺货、库存预警、高风险供应商、延迟路线四类运营指标。说明系统会给出处理建议，并按库存、供应商、物流、成本四个维度展开明细。

## 4. 风险中心

展示 Critical、High、Medium、Low 风险数量，以及待处理风险卡片。强调每条风险包含关联对象、原因描述和处置建议。

## 5. AI 工作台

建议问题：

- 本周供应链最大的风险是什么？
- 哪些商品未来一周可能缺货？
- 哪些供应商需要重点关注？
- 请生成一份供应链运营简报。

说明 Python AI 服务可接入真实 LLM；未配置 Key 时使用规则模式，仍会返回结构化结论、证据和建议。

## 6. 降级演示

关闭 MySQL 或 Python AI 服务后刷新页面，接口仍返回成功响应。重点展示 `metadata.source`、`metadata.fallbackReason` 和服务状态组件。
